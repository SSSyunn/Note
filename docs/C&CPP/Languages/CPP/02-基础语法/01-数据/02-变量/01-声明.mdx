## 一、基本类型声明形式

一个简单变量的声明形式即：类型说明符 + 变量名

```cpp
bool flag;
char ch;
int ival;
double dval;
```

- 当变量在程序中首次声明时，通常会隐式进行定义和初始化，即创建一个内存空间用于保存该变量并为其赋予初始值

- 在创建变量时可以显式赋予初始值，如：

  ```cpp
  bool flag = true;
  char ch = 'A';
  int ival = 48;
  double dval = 3.14;
  ```

  如果不显式赋予变量初始值，变量也会在创建时获得一个初始值，对于内置类型来说，该初始值并不固定，而是一个垃圾值。具体初始值取决于该变量所在的位置（初始化形式），参考【[初始化-初始化形式](../初始化/#二初始化形式)】



## 二、复杂类型声明形式

### 1. 简单复合类型声明

```cpp
int a = 10;
int &ra = a;
int &ra2 = ra;

int *p;
int *pn = nullptr;
int *pa = &a;

int arr1[5];
int arr2[5] = { 1, 2, 3, 4, 5 };
int arr3[5] = { 1, 2, 3 };
int arr4[] = { 1, 2, 3 };
int str[] = "str";
int str[] = { 's', 't', 'r' };
```

> 引用、指针与数组的初始值相关内容见【[初始化-初始值](../初始化/#一初始值)】



### 2. 复杂类型声明

由于复合类型的存在，所以可能会出现一些特别复杂的变量类型，例如简单一点的包括 "指针的指针"、"数组的指针"、 "数组的引用"、"指针的数组" 等等。

更复杂的类型还有 "指针数组的指针"、"指针数组的引用" 等等，甚至一些可能并不会用到但却允许存在的类型，这些类型的变量在声明时会遇到错综复杂的符号现象，分析这类声明的一个重要方法就是先找到变量名，然后根据运算符的优先级一层一层往外分析，例如：

```cpp
int **(*arrayPtr)[10];
```

我们按照上述方法对该类型进行分析：

- 首先找到变量名`arrayPtr`，然后根据运算符的优先级找到最优先与它匹配的一个运算符，即括号内的`*`，这个符号表明这个变量是一个指针。

  > 注意此处括号的重要性，由于`[]`运算符比`*`运算符的优先级高，所以如果不添加括号，那么这个变量将优先和`[]`运算符结合，从而表明该变量是一个数组而非指针

- 再将`(*arrayPtr)`看作一个整体，它优先和`[10]`匹配，表明这是一个数组。首先我们对语义进行分析，`arrayPtr`首先是一个指针，它应该指向一个数组；其次从形式上分析，`(*arrayPtr)`是一个数组，也就是对变量进行解引用得到的是一个数组。

- 我们已经可以将`(*arrayPtr)[10]`看作一个整体了，对一个数组进行`[]`运算符操作，得到的就是其中一个元素，所以剩下的部分`int **`就是这个数组的元素类型，我们可以知道这个元素类型实际上就是指向指针的指针

- 综上所述，上述声明得到了一个变量`arrayPtr`，这是一个指向拥有 10 个元素数组的指针，数组的每个元素是指向`int`指针的指针



## 三、多变量统一声明

可以在一条声明语句中声明多个变量，前提是声明这些变量所需要的类型说明符相同：

```cpp
bool flag1, flag2;
int ival1, ival2 = 48;
double dval1 = 3.14, dval2 = 2e10;
```

- 同一条声明语句中的多个变量之间用逗号分隔
- 定义时可为一个或多个变量赋予初始值

也可以在一条声明语句中同时声明基本类型变量和复合类型变量：

```cpp
int ival, *pi = nullptr, &ri = ival, arr[10];
```

- 和前面相同，只要满足基本要求即放在最前面的类型说明符相同即可
- 复合类型变量在声明时，声明符（`*`、`&`、`[]`）和变量名是一体的，但是我们在描述一个复合类型变量的类型时，却应该把这些声明符和其根类型看作是一体的，即（`int*`，`int&`，`int[10]`）等
